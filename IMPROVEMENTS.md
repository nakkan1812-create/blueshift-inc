## 0. 大前提（改善の鉄則）

4つのAIが揃って褒めた武器は **①スピード ②透明な価格 ③住友商事20年のリアルな知見**。
信頼性を足そうとして情報を増やすと、**重く・遅そうに見えて武器を殺す**罠がある。

- **鉄則: 武器を消さず、不安だけ消す。**
- **盛らない・捏造しない。** この会社のコアバリュー（建前でなくリアルな本音を教える）と
  矛盾する誇張は、バレた瞬間に最大の資産＝信用を失う。正直なまま信頼を作る。

## 2. CEO（中尾）の方針決定 — 確定事項

このセクションは創業者本人が決めた確定方針。実装時はこれを優先する。

### 2-1. 納期: 3–5営業日 → **5–10営業日** に変更
- **理由**: 40–60ページを人間が本当に練ったなら3–5日は逆に「AIが吐いただけ」の証拠に見える。
  短納期すぎると逆効果。5–10日が「速いが、ちゃんと人が考えている」絶妙ライン。
- **棚卸し対象（要修正箇所）**:
  - [ ] サイト全ページの納期表記（`why.html` / `sample.html` / `brief.html` / `index.html` 等）
  - [ ] **会社ロゴ**内の納期表記（あれば）
  - [ ] **LinkedIn** プロフィール / 投稿の納期表記
  - [ ] 名刺 `card.html` の納期表記（あれば）

### 2-2. 価格: Standard のドル表記を **$5,000** に統一
- 中途半端だった金額をキリよく。$5,000 を基準表記、¥は為替で都度調整。
- [ ] 全ページのプラン表で統一（`why.html` / `sample.html` / `brief.html`）

### 2-3. 業界フォーカス（黒からも指摘あり）
- **強い3〜5領域**を明示し「何でも屋」に見せない。
- 外れる案件は **事前要件の段階で24時間以内に正直にお断り** → 誠意＝信用。
- [ ] フォーカス領域の確定（例: Enterprise Software / Energy / Manufacturing / Retail など — 中尾の実ネットワークに基づき確定）

### 2-4. 一人会社 → 体制と品質保証の可視化
- **方針: 採用予定者を現headcountに混ぜて「3名」と書くのは盛りリスク**（突っ込まれると弱く、
  「盛らない」原則と矛盾）。正直フレームを推奨:
  - 実体 = **創業者 ＋ 専門アドバイザー網（商社OB等）**
  - 採用は **「Scaling: 2026年中に専門人材を増員予定」** と “予定として” 明記し現体制と混ぜない
- **品質保証の可視化（積極的にやる）** — “絞っている＝丁寧” の証明:
  - [ ] 「品質維持のため**同時受注は ◯件まで**」（◯は中尾が決定）
  - [ ] 「全レポート**二段階レビュー**」等の品質管理プロセス明記
- ※「3名」と打ち出すか最終判断は中尾。打ち出す場合も「創業者＋採用予定2名」と内訳を正直に。

### 2-5. プラン内訳の明示 + 「Starterで終わらせない」設計
- 各プラン（Starter / Standard / Premium）に**何が含まれるか**を表で明示。
- **アップグレード階段**を設計に組み込む:
  - Starter = 味見（小さく品質確認）
  - → Standard = 本命（競合マップ・パートナー候補・収益シナリオ）
  - → Premium = 温かいパートナー紹介・実行支援
- [ ] 各プランの含有物テーブルを作成し、上位への動線（次に何が増えるか）を明記

### 2-6. サンプルケースの拡充（Notion以外も）
- 現状の強い資産: `japan_market_entry_demo_notion.html`（Notion風）/ `estimate_tool_demo.html`
- **Notion以外の形式を2〜3追加** → 実績代わりの厚みを出す:
  - [ ] PDFエグゼクティブサマリー版
  - [ ] 業界別ミニケース（フォーカス領域に対応させると 2-3 と連動）
  - [ ] （任意）1ページ・スナップショット版

### 2-7. ★法人登記と社名表記（「Inc.」問題）— 最優先で潰す
- **背景**: 現状「Blueshift **Inc.**」を名乗っているが**未登記**。「Inc.」は登記された法人(corporation)を
  意味するため、未登記での使用は **会社法7条（会社と誤認される表記の禁止／100万円以下の過料）** に
  抵触しうる。海外取引先への誤認表示リスクもあり、「盛らない」コアバリューとも矛盾する。
- **CEO決定: 社名を「Blueshift LLC」または「Blueshift Japan」に変更する。**
  - 「Blueshift LLC」を名乗るなら **合同会社の登記が前提**（"LLC" も法人表記のため）。
  - 「Blueshift Japan」は法人表記を含まないため、登記前でも比較的使いやすい（屋号的運用も可）。
    ただし将来的には合同会社化を推奨。
  - **どちらに転んでも "Inc." はサイト・ロゴ・LinkedIn・名刺から全削除する。**
- **コスト目安（合同会社の場合）**:
  - 設立: 実費 **約6〜10万円**・最短数日〜2週間
  - 維持: **年7万円〜**（法人住民税 均等割の最低額）。自分で記帳なら年7〜13万円、
    税理士に依頼で年22〜40万円。役員報酬を取ると社会保険が別途発生し得る。
  - 合同会社は決算公告不要・役員任期なし（重任登記不要）で株式会社より維持が軽い。
- **棚卸し対象（"Inc." → 新名称へ一括置換）**:
  - [ ] 全 *.html の社名表記
  - [ ] `favicon.svg` / ロゴ画像
  - [ ] `card.html`（名刺）
  - [ ] LinkedIn / 各種プロフィール
  - [ ] リポジトリ名 `blueshift-inc` / ドメイン `blueshift-inc.jp`（変更要否を検討）
  - [ ] `CNAME`（ドメイン変更する場合）
- ※ 登記要否・税務・社会保険の最終確認は **司法書士・税理士** に相談すること（私は法律家ではない）。
